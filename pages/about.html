À propos du projet et des données

Objectif du projet

Obesity Insights est un tableau de bord interactif réalisé dans le cadre d’un projet universitaire de visualisation de données. L’objectif est d’explorer l’obésité chez les adultes aux États-Unis à partir d’un jeu de données réel, en combinant plusieurs points de vue :

  • l'évolution géographique de l'obésité (carte par État),
  • les comportements de santé associés (alimentation, activité),
  • la dimension démographique (âge, genre),
  • les facteurs socio-économiques (revenu, éducation, origine ethnique).

Le site a été conçu pour être à la fois visuel, interactif et pédagogique : chaque page explique le choix des graphiques et la manière d’interpréter les résultats.

Structure du jeu de données

Le jeu de données provient d’enquêtes de santé répétées chaque année. Chaque ligne représente un groupe de personnes (par exemple : “Adultes de 35–44 ans, femmes, dans l’État de l’Alabama, en 2011”) et donne la proportion d’adultes répondant à une question précise.

Les principaux champs utilisés dans ce projet sont :

  • YearStart / YearEnd : année de référence des données. Dans les visualisations, nous utilisons surtout YearStart.
  • LocationAbbr : abréviation de la zone géographique (par exemple AL pour l’Alabama, US pour l’agrégation nationale).
  • LocationDesc : nom complet de l’État ou de la zone (par exemple “Alabama”, “United States”).
  • Question : texte complet de la question posée, par exemple “Percent of adults aged 18 years and older who have obesity”. Le site se concentre principalement sur cette question et quelques autres liées à l’alimentation et à l’activité physique.
  • QuestionID : identifiant court de la question (par exemple Q036 pour l’obésité, Q018 pour la consommation de fruits, Q019 pour les légumes, Q047 pour l’activité de loisir). Cet identifiant est utilisé dans le code pour filtrer les indicateurs pertinents pour chaque visualisation.
  • Data_Value : valeur numérique principale du jeu de données. Il s’agit d’un pourcentage : la proportion d’adultes du groupe concerné qui présentent la caractéristique mesurée (par exemple “sont obèses”, “ne mangent pas de fruits tous les jours”, “n’ont aucune activité de loisir”).
  • StratificationCategory1 et Stratification1 : ces deux champs décrivent la stratification utilisée pour la ligne. Par exemple :
    • StratificationCategory1 = "Age (years)" Stratification1 = "35 - 44"
    • StratificationCategory1 = "Sex" Stratification1 = "Male" ou "Female"
    • StratificationCategory1 = "Education" ou "Income"
    Dans ce projet, nous utilisons surtout ces champs pour créer les comparaisons par âge et par genre, ou pour filtrer les données sur le total (“Total / Total”).
  • GeoLocation : coordonnées géographiques (latitude, longitude) de la zone. Ce champ pourrait servir à d’autres types de cartes, mais pour la carte principale nous utilisons un fond de carte standard des États-Unis.

Comment les visualisations utilisent ces champs ?

  • Carte choroplèthe : agrège les lignes par (YearStart, LocationDesc) pour un indicateur unique (QuestionID = Q036, obésité), puis colorie chaque État en fonction de Data_Value.
  • Radar activité / alimentation : pour chaque (État, année), récupère plusieurs questions (Q036, Q018, Q019, Q047) et construit un profil combiné à partir de leurs pourcentages.
  • Démographie : utilise les mêmes indicateurs d'obésité mais en exploitant StratificationCategory1 = "Age (years)" ou "Sex", afin de comparer les tranches d'âge et les genres au niveau national ou pour un État particulier.
  • Facteurs socio-économiques : explore les disparités en utilisant StratificationCategory1 = "Income", "Education", ou "Race/Ethnicity", révélant comment l'obésité est liée aux conditions sociales et économiques à travers des barres horizontales et des lollipop charts.

L’ensemble de ces choix permet de montrer que derrière un taux d’obésité global se cachent des profils très différents selon les territoires, les comportements de santé et les caractéristiques démographiques.
